# スライドアイデア

## 全体構成

### 第1部: MCPの理解
#### スライド1: タイトルスライド
- タイトル: LINEボットでMCPサーバーを活用する〜GitHub Codespaces開発ガイド
- 対象者: MCPに興味がある、AIエージェントの仕組みを理解したい

#### スライド2: MCPとは
- Model Context Protocol (MCP) の定義
- MCPのアーキテクチャ
  - ホスト（VS Code、Claude Desktopなど）
  - クライアント（ホストに組み込まれたMCP接続管理部分）
  - サーバー（ツール、リソース、プロンプトを提供）
- MCPが解決する課題：AIエージェントが外部ツール・リソースに安全かつ構造化されたアクセス

#### スライド3: MCPの利点
- 開発効率向上：新しいツールの実装が簡単
- セキュリティ：認証・認可の標準化
- スケーラビリティ：複数の外部リソースを一括管理
- 相互運用性：様々なAIエージェントで利用可能（GitHub Copilot、Claude、Cline等）

#### スライド4: MCPの実例
- Azure Data ExploreでのMCP活用
- App ServiceをMCPサーバーとして公開
- Window Package Manager（WinGet）のMCP対応
- Fabricリアルタイムインテリジェンス（RTI）

### 第2部: MCPサーバー開発
#### スライド5: MCPサーバーの構成要素
- **ツール**: AIエージェントが呼び出す関数（入出力がスキーマで定義）
- **リソース**: テキストやJSONドキュメントなど（参考情報の提供）
- **プロンプト**: AIエージェントへの指示テンプレート
- **サンプル**: 天気予報bot、データベースクエリツール等

#### スライド6: MCPサーバー開発の流れ
1. MCPサーバーのフレームワーク選択（Python, Node.js等）
2. ツールの定義
   - 関数シグネチャ
   - パラメータスキーマ
   - 戻り値の型
3. LLMとの連携実装
4. ローカルテストと検証
5. デプロイ

#### スライド7: 認証とセキュリティ
- API キー管理
- OAuth認証のサポート
- MCP container化による分離
- ログとオーディット

## 補足：各セクションのポイント

### MCPセクションの強調点
- MCPは単なるAPIではなく、「構造化された安全な相互作用」を実現するプロトコル
- 従来のAPI連携との違いを明確化

### 開発セクションの強調点
- 最小限で動作するサンプルから始める
- 段階的な複雑化

### LINE連携セクションの強調点
- LINEのユーザーフレンドリー性とMCPの組み合わせ
- AIエージェントの実際の活用場面

### Codespeaces活用セクションの強調点
- 環境構築の簡素化
- チーム開発・学習効率の向上
- 初心者〜経験者まで対応可能な開発環境

### ハンズオンセクションの強調点
- 「作ってみる」ことで理解が深まる
- 実際に動作するサンプルコードの提供
